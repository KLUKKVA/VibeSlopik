# Desktop Host

Скачайте архив для Windows x64, Linux x64, macOS Intel или Apple Silicon,
распакуйте в постоянную папку и запустите `VibeSlopik-Host`. Для релизного
архива Python не нужен.

Без аргументов откроется меню. При первом запуске Host проверит Codex, папку
проектов, URL и данные Relay и режим совместимости. Конфигурация атомарно
хранится в `data/config.toml`, секреты отдельно в `data/secret.env`. Из меню
доступны диагностика, кэш, язык, автозапуск и Speech Pack.

Если синтаксис `config.toml` повреждён, Host сохраняет исходник как
`config.toml.corrupt-TIMESTAMP`, восстанавливает безопасные значения и явно
сообщает об этом. Смысловые ошибки не затираются молча: диагностика указывает
конкретную неверную настройку.

Выберите **Запустить Host** в меню (или выполните `VibeSlopik-Host start`). Host
работает в фоне и пишет журналы в `data/logs`. Для штатной авторизованной
остановки выберите **Остановить Host** или выполните `VibeSlopik-Host stop`.
Также доступны команды `status`, `logs` и `diagnostics`; диагностический отчёт
не содержит токены и переписки. Команда `run` оставлена для запуска в переднем
плане и системных служб.

Во время настройки Host реально запускает `codex --version`. Если Windows
подставляет недоступный псевдоним из WindowsApps, установите Codex CLI или
укажите путь к запускаемому `codex`: нерабочий путь не будет сохранён.
