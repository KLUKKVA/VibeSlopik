# Архитектура

iOS отправляет авторизованный JSON в `/v1/client/{hostId}` на Relay. Desktop
Host держит long-poll `/v1/host/{hostId}/poll`, передаёт запрос своему локальному
HTTP API и возвращает ответ. Relay не подключается к компьютеру входящим
соединением, поэтому NAT и динамический домашний IP не мешают.

Go Relay ограничивает тело, очереди, число ожиданий и таймауты. Регистрация
атомарно сохраняется с резервной копией. Host использует idempotency ID, хранит
ограниченную очередь событий Codex и подтверждает появление turn до удаления
черновика iPhone.

Пути к фото не покидают компьютер. Host превращает пути из авторизованного чата
в непрозрачные ID, создаёт миниатюру 480 px, а оригинал загружается только при
открытии просмотрщика. Подробные endpoint описаны в [`protocol.md`](protocol.md).
