# Glossary

- **Codex app-server:** the local Codex JSON-RPC process controlled by Host.
- **Host:** the desktop process that owns Codex access and project files.
- **Relay:** the small VPS rendezvous service; it does not run Codex.
- **Host ID:** public identifier selecting one registered Host on Relay.
- **Host secret:** credential used only by Host when polling Relay.
- **Client token:** credential entered into the iOS application.
- **Relay admin key:** VPS credential used to register or update a Host.
- **Thread/chat:** one Codex conversation associated with a project directory.
- **Turn:** one user request and the resulting Codex work.
- **Speech Pack:** optional local faster-whisper runtime and model.
- **Compatibility mode:** policy controlling feature probes after Codex changes.
